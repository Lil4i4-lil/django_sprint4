## 📋 О проекте

**Блогикум** — это современная веб-платформа для создания и управления персональными блогами. Проект построен на лучших практиках Django и предоставляет пользователям удобную среду для публикации записей, взаимодействия с сообществом и кастомизации личного пространства.

## 🚀 Ключевые возможности

### 👤 **Пользователи**
- 🔐 Регистрация и безопасный вход в личный кабинет
- 🖼️ Возможность настройки профиля

### 📄 **Управление публикациями**
- ➕ Создание новых постов с заголовком, текстом и изображениями
- ✏️ Редактирование существующих записей
- 🗑️ Удаление публикаций
- 🏷️ Добавление категорий к постам для удобной навигации

### 💬 **Взаимодействие**
- 📝 Комментирование записей
- 🔎 Фильтрация и поиск по категориям
- 📊 Пагинация для удобного просмотра записей

### 🎨 **Интерфейс**
- 📱 Адаптивный дизайн на Bootstrap 5
- 🖥️ Интуитивно понятный интерфейс
- ⚡ Быстрый доступ ко всем функциям
- 💬 Система уведомлений об успешных действиях

## 🛠️ Технологический стек

| Компонент | Технология                |
|-----------|---------------------------|
| **Backend** | Python 3.10+, Django      |
| **База данных** | SQLite                    |
| **Аутентификация** | Стандартная модель Django |
| **Frontend** | HTML5, Bootstrap 5        |
| **Шаблонизация** | Django Templates          |
| **Тестирование** | pytest                    |

## 📁 Структура проекта

| Приложение | Описание | Ключевые модели                   |
|:-----------|:--------|:----------------------------------|
| **blog** | 📰 Основное приложение для управления постами и комментариями | Post, Category, Location, Comment |
| **pages** | 📄 Статические страницы и обработка ошибок | -                                 |
| **registration** | 🔐 Аутентификация и регистрация | -                                 |

## 🗄️ База данных

Проект использует гибкую систему управления данными:
- Пользователи хранятся в стандартной модели Django
- Посты привязаны к авторам и категориям
- Комментарии связаны с постами
- Оптимизированные запросы для предотвращения N+1 проблем

## 🖼️ Медиафайлы

Проект поддерживает загрузку изображений:
- Изображения для постов хранятся в `media/posts_images/`

## 🧪 Тестирование

В проекте реализовано комплексное тестирование с использованием **Pytest**.
Директория tests/ содержит все тесты проекта.\
В корне проекта хранится конфигурационный файл **pytest.ini**

## 📦 Управление зависимостями

Проект использует стандартный pip с файлом `requirements.txt` для управления зависимостями. Все необходимые пакеты и их версии фиксируются для обеспечения воспроизводимости сборки.
